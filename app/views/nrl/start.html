Initial designs

These prototypes explore the initial ideas floated for onboarding the National Record Locator service within the redesigned Summary Care Record application. NB - the layout used in this prototype is still in development and likely to change.

We explored the possibility of linking out to more detailed record information from the patient overview 'dashboard', and designed what this would look like in 3 different scenarios:

  • Few records
  • Many records
  • No records

Hypotheses

If we allow users to view available care plans on the overview page

Then they will be able to quickly recognise whether or not the user has any exisiting care plans/documentation

Because the information they need is available readily as soon as they reach the overview page.


If we provide links to detailed information about a patient's care plan(s) on the overview page

Then users will be able identify they information they need from a non-cluttered interface

Because the majority of the information will be stored in a separate page, linked to via the overview page.
