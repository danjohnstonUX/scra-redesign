About this prototype

This prototype explores the introduction of edit functionality within the redesigned SCRa.

This feature allows those users with the correct RBAC permissions to edit a patient's demographics details. The ability to edit details exists in existing SCRa, so we are reimplementing it using the new UI.

Version 2 prototypes

We have revisited the initial designs from November of 2019 and updated them with feedback from both users and clinical safety sessions ready to re-test ahead of development.

We aim to test navigation within the 'Edit' screen as users told us they wanted an up-front view of all the sections that were available to edit for a particular area of the patients details, as well as the ability to cancel in progress amendments and remove previous amendments.

Version 2 hypotheses

If we add the editable sections to our navigation components

Then users will be able to see which details are avaiable to edit as soon as they land on the page, and can then subsequently navigate to the relevant section

Because our navigation components are always visible.


If we provide a 'cancel' button for users who have accidentally chosen to add new information to a patient's record

Then users will be confident that they have not made any accidental, unnecessary changes to a patient's PDS record

Because the 'cancel' button will allow them to stop what they are doing and return to the previous state.


If we provide a 'remove' button for previously entered information which can be removed

Then users will be able to quickly amend old, unnecessary or out of date information without having to manually delete information using their keyoard.

Because the 'remove' button will perform this action for them.


If we ask users to confirm their changes after clicking the 'Save' button at the bottom of the Edit page

Then users will be confident that they have not made any mistakes when amending a patients' data

Because we show the users what the information has changed from and to before they commit to saving the information on Spine, and allow them to cancel any changes that do not look correct.
